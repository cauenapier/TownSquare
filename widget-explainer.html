1 · What the widget is

A host page imports public/townsquare.mjs and calls mountTownSquare(rootEl, options). That single call owns everything inside the mount root: scene rendering, keyboard/pointer input, a local animation loop, chat bubbles, and one WebSocket connection to a TownSquare server. It returns a handle with updateConfig(), destroy(), and a live ctx.

The implementation is split by concern under public/widget/; the mount file is just an orchestrator. All per-mount state lives in a single context object (ctx) that is threaded through every module — there is no hidden per-mount state hiding in module scope.

5 · Lifecycle & teardown

destroy() is thorough and worth using as the checklist of everything a mount owns: it sets ctx.disposed = true (which short-circuits the rAF loop and the reconnect handler), then unwinds in reverse: stopGameLoop → destroy birds/clouds/plugins → unwire keyboard, stage pointer, help panel → teardown connections & message board → remove button listeners → unwatch page (restoring the patched history.pushState/replaceState) → clear reconnect/typing timers → socket.close() → root.replaceChildren().

Mental model: mount wires N subsystems onto one ctx; destroy must unwire all N. Any new addEventListener, timer, or global patch added during mount is a teardown obligation — that symmetry is the thing to protect when refactoring.
